### В этом репозитории собраны пет-проекты для практики программирования и тестирования с полной документацией. 
Цель репозитория - демонстрация навыков тестирования и программирования для портфолио QA-инженера.
---

## 📂 Структура репозитория
Каждый проект оформлен как отдельная папка с собственным `README.md` и включает три основные подпапки:

| Папка           | Содержание |
|-----------------|------------|
| `src/`         | Исходный код проекта (`*.py`) |
| `bug reports_/`         | Баг-репорты (`bug_reports.md`) |
| `tests/`         | Тестовая документация: <br>• `test_plan.md` <br>• `test_cases.md` <br>• `checklist.md` <br>• `test_report.md` |

---

##  Список проектов на данный момент

| Проект | Описание |
|--------|----------|
| Числовая угадайка | Игра с угадыванием случайного числа, подсказками и подсчетом попыток |
| Игра «Виселица» | Классическая игра с угадыванием слова по буквам |
| Генератор паролей | Генерация безопасных паролей с разными символами и длиной |
| Шифр Цезаря | Шифрование и дешифрование текста с русским и английским алфавитом |
| Конвертер систем счисления | Конвертация чисел между десятичной, двоичной, восьмеричной и шестнадцатеричной |
| Магический шар | Игра с предсказаниями на основе случайного выбора |

## Направления развития репозитория

### Тест-дизайн
- Приведение всех тест-кейсов к единому формату для удобства поддержки  
- Расширение спектра техник тестирования и методов тест-дизайна:  
  - Pairwise Testing  
  - Decision Table Testing  
  - State Transition Testing  
  - Performance и Load Testing  

### Новые сущности и инструменты
- Добавление тестирования новых сущностей: API и GUI  
- Демонстрация работы с инструментами (например, коллекции Postman для API)  

### Автоматизация и новые проекты
- Введение автоматизации тестирования по мере изучения инструментов  
- Добавление новых проектов и экспериментальных сценариев  
- Улучшение документации и примеров использования
