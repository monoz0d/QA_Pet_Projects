# Баг репорты - Магический шар

## BUG-001 - Приветствие: некорректный вывод при пустом имени

## Подробное описание  
При запуске программы пользователь может оставить имя пустым, нажав Enter.  

Согласно требованиям проекта и тест-кейсу **MB-004**, программа должна корректно обрабатывать пустой ввод имени.

---

### Требование  
Программа должна:
- обрабатывать пустой ввод имени  
- выводить нейтральное приветствие без использования пустых значений  

---

### Ожидаемый результат  
Программа выводит корректное приветствие.  

**Пример:**  
```text
Привет!
```
### Фактический результат
```text
Привет,  
``` 
## Шаги воспроизведения
- Запустить программу
- На запрос имени нажать Enter
 ## Окружение 
- Среда: Dev / Staging
- Язык: Python 3.14
- Тип приложения: консольное
## Severity

Низкая

### Обоснование

Дефект не влияет на основную функциональность программы.

## Priority

Средняя

### Обоснование

Влияет на пользовательский опыт.

## BUG-002 - Логика продолжения: завершение программы при пустом вводе
## Подробное описание

При ответе на вопрос о продолжении пользователь может ввести пустую строку.

Согласно требованиям проекта и тест-кейсу **MB-008**, программа должна валидировать ввод и повторно запрашивать корректное значение.

### Требование

Программа должна:

- принимать только значения да или нет
- при пустом вводе повторно запрашивать ответ
- не завершать выполнение программы
### Ожидаемый результат
- Программа повторно запрашивает ввод
- Программа продолжает работу
### Фактический результат
- Программа завершает работу.
## Шаги воспроизведения
- Запустить программу
- Ввести имя
- Задать любой вопрос
- На вопрос о продолжении нажать Enter
## Окружение (Environment)
- Среда: Dev / Staging
- Язык: Python 3.14
- Тип приложения: консольное
## Severity

Средняя

### Обоснование

Нарушается логика работы программы.

## Priority

Высокая

### Обоснование

Пользователь может случайно завершить программу

- Тип тестирования: негативный

## BUG-003 - Логика продолжения: завершение программы при некорректном вводе
## Подробное описание

При вводе произвольного значения вместо да или нет программа завершает работу.

Согласно требованиям проекта и тест-кейсу **MB-009**, система должна корректно обрабатывать некорректный ввод.

### Требование

Программа должна:

- валидировать пользовательский ввод
- принимать только значения да или нет
- при некорректном вводе повторять запрос
### Ожидаемый результат
- Программа повторно запрашивает ввод
- Программа продолжает работу
### Фактический результат

- Программа завершает работу.

## Шаги воспроизведения
- Запустить программу
- Ввести имя
- Задать любой вопрос
- Ввести "не знаю"
- Нажать Enter
## Окружение (Environment)
- Среда: Dev / Staging
- Язык: Python 3.14
- Тип приложения: консольное
## Severity

Средняя

### Обоснование

Нарушена логика обработки пользовательского ввода.

## Priority

Высокая

### Обоснование

Высокая вероятность ошибки пользователя может привести к ухудшению пользовательского опыта

- Тип тестирования: негативный
