# Требования: "Генератор паролей"

---

## 1. Введение

### 1.1 Описание
Приложение реализует генерацию случайных паролей с настройкой длины, набора символов (цифры, строчные/прописные буквы, символы) и исключением неоднозначных символов.  

---

## 2. Функциональные требования

### FR-001: генерация пароля
**Описание:** Система должна генерировать пароль по заданным параметрам.  
**Входные данные:**  
- Количество паролей (целое число)  
- Длина пароля (целое число)  
- Выбор символов (цифры, строчные, прописные, спецсимволы)  
- Исключение неоднозначных символов (логическое значение)  
**Выходные данные:**  
- Список сгенерированных паролей

### FR-002: исключение неоднозначных символов
**Описание:** Система должна исключать символы `il1Lo0O` при включении опции.  

### FR-003: обработка некорректного ввода
**Описание:** Система должна проверять корректность ввода и запрашивать повторный ввод при ошибках.  

---

## 3. Варианты использования

### UC-001: генерация одного пароля
**Актор:** пользователь  
**Сценарий:**  
1. Запуск программы  
2. Ввод параметров генерации (кол-во, длина, группы символов)  
3. Получение результата  

### UC-002: генерация нескольких паролей
**Актор:** пользователь  
**Сценарий:**  
1. Запуск программы  
2. Ввод параметров генерации  
3. Получение нескольких паролей  

### UC-003: некорректный ввод
**Актор:** пользователь  
**Сценарий:**  
1. Ввод некорректных данных (например, буквы вместо числа, неправильный выбор да/нет)  
2. Программа выводит сообщение об ошибке и повторно запрашивает ввод  

---

## 4. Пользовательские истории
- Как пользователь, я хочу генерировать пароли разной длины, чтобы использовать их в разных сервисах.  
- Как пользователь, я хочу включать или исключать различные группы символов, чтобы повышать безопасность пароля.  
- Как пользователь, я хочу исключать неоднозначные символы, чтобы легко различать символы.  
- Как пользователь, я хочу получать сообщение об ошибке при некорректном вводе, чтобы корректно ввести данные.  

---

## 5. Нефункциональные требования (NFR)

### NFR-001: надежность
Система не должна аварийно завершаться при ошибках ввода.  

### NFR-002: производительность
Система должна генерировать пароли любой длины до 100 символов без заметной задержки.  

### NFR-003: удобство использования
Интерфейс должен быть понятным (консольный ввод с подсказками).  

### NFR-004: валидация
Система должна проверять корректность входных данных.  

---

## 6. Ограничения
- Программа работает только в консоли  
- Нет графического интерфейса  
- Поддерживаются только стандартные группы символов (цифры, строчные/прописные буквы, спецсимволы)  
- Нет интеграции с внешними системами  

---
