# Тест-план проекта "Генератор паролей"

## 1. Введение
**Описание проекта:**  
Проект реализует генератор случайных паролей на Python с возможностью настройки состава символов. 
**Поддерживаемые функции:** 
- Выбор количества генерируемых паролей
- Выбор включения цифр, строчных и прописных латинских букв и специальных символов
- исключение неоднозначных символов (`il1Lo0O`).

**Цель тестирования:**  
Обеспечить корректную генерацию паролей заданной длины и количества, проверить корректность включения/исключения различных групп символов, а также устойчивость программы к некорректному вводу.

**Особенности тестирования:**  
- Фокус на функциональности генерации пароля  
- Проверка работы с различными комбинациями символов  
- Негативные тесты на некорректный ввод  

---

## 2. Область тестирования
**Покрываемые модули:**  
- Модуль ввода количества паролей и длины пароля  
- Модуль выбора включаемых символов  
- Алгоритм генерации пароля (`generate_password`)  
- Функция проверки корректности ввода (`verify`)  

**Исключаем:**  
- Работа с внешними файлами  
- Сетевые операции и API  
- UI-тестирование (консольная программа)  

---

## 3. Тестовая стратегия
**Типы тестирования:**  
- **Функциональное:** проверка корректной генерации пароля заданной длины и с выбранными символами  
- **Негативное тестирование:** проверка реакции на некорректный ввод (не `да`/`нет`, пустой ввод)  
- **Граничные значения:**  
  - Длина пароля = 1  
  - Длина пароля = максимально допустимая (например, 100)  
  - Количество паролей = 1  
  - Количество паролей = большое число (например, 1000)  
- **Исследовательское тестирование:** выявление неожиданных ошибок при сочетании всех опций  

**Приоритет тестов:**  
1. Критические: корректная генерация пароля, падение программы при некорректном вводе  
2. Средний: обработка всех групп символов и исключение неоднозначных символов  
3. Низкий: производительность при генерации большого числа паролей

## 7. Риски и ограничения
**Риски:**  
- Падение программы из-за некорректного пользовательского ввода
- Потеря символов при исключении неоднозначных символов  
- Невозможность сгенерировать пароль при пустом наборе символов 

**Ограничения:**  
- Нет UI/UX тестирования, т.к. проект консольный
- Нет интеграции с внешними системами или базами данных
