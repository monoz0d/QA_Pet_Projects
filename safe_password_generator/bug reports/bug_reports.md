# Баг репорты — Генератор паролей

## BUG-001  
### Пустой набор символов (NG-010)

### Окружение
- Dev / Staging  
- Python 3.14  
- Консольное приложение  

### Связанные артефакты
- Тест-кейс: NG-010  
- Чек-лист: негативные сценарии (пустой набор символов)  
- Тип тестирования: негативное  

### Шаги воспроизведения
1. Запустить программу  
2. Не включать ни одну группу символов  
3. Попытаться сгенерировать пароль  

### Ожидаемый результат
- Программа не падает  
- Сообщение об ошибке  
- Запрос корректного выбора символов  

### Фактический результат
- Программа завершает работу с ошибкой:  
```IndexError: Cannot choose from an empty sequence```  

### Severity
Высокий  

### Priority
Высокий  

### Обоснование

**Severity:** ошибка блокирует основной функционал - генерацию паролей.

**Priority:** баг легко воспроизводится и влияет на пользовательский опыт.

## BUG-002  
### Ввод букв в длину пароля (NG-011)

### Окружение
- Dev / Staging  
- Python 3.14  
- Консольное приложение  

### Связанные артефакты
- Тест-кейс: NG-011  
- Чек-лист: негативные сценарии (ввод букв вместо числа)  
- Тип тестирования: негативное  

### Шаги воспроизведения
1. Запустить программу  
2. Ввести количество паролей: 1  
3. Ввести длину пароля: `test`  
4. Нажать Enter  

### Ожидаемый результат
- Программа не падает  
- Сообщение о некорректном вводе  
- Повторный запрос ввода  

### Фактический результат
- Программа завершает работу с ошибкой:  
```ValueError: invalid literal for int() with base 10: 'test'```  

### Severity
Высокий  

### Priority
Высокий  

### Обоснование

**Severity:** ошибка приводит к падению программы при любом некорректном вводе длины пароля, блокируя возможность генерации паролей.  

**Priority:** баг легко воспроизводится пользователями, напрямую связан с основным функционалом программы и требует немедленного исправления.
