# Тест-план проекта "Caesar Cipher"

## 1. Введение
**Описание проекта:**  
Проект реализует шифр Цезаря для английского и русского алфавитов с возможностью шифрования и дешифрования текста. Программа поддерживает:  
- Один шаг сдвига и диапазон сдвига  
- Сохранение регистра букв  
- Игнорирование цифр и спецсимволов  

**Цель тестирования:**  
Обеспечить корректное шифрование и дешифрование текста, проверить обработку граничных случаев, исключить падения программы при некорректном вводе.  

**Особенности тестирования:**  
- Фокус на функциональности текстового алгоритма  
- Проверка работы с разными алфавитами и диапазонами сдвига  
- Негативные тесты на пользовательский ввод  

---

## 2. Область тестирования
**Покрываемые модули:**  
- Модуль ввода текста  
- Модуль выбора языка  
- Алгоритм шифрования и дешифрования  
- Обработка диапазона сдвига  

**Исключаем:**  
- Работа с внешними файлами  
- Сетевые операции и API (проект локальный)  
- UI-тестирование (консольная программа)  

---

## 3. Тестовая стратегия
**Типы тестирования:**  
- **Функциональное:** проверка корректного шифрования и дешифрования текста  
- **Негативное тестирование:** проверка реакции на некорректный ввод, пустую строку, спецсимволы  
- **Граничные значения:** шаг сдвига = 0, шаг = длина алфавита, пустой текст  
- **Производительность:** проверка работы с очень длинным текстом  
- **Исследовательское тестирование** для выявления более широкого спектра дефектов 

**Приоритет тестов:**  
1. Критические: корректное шифрование/дешифрование, падение программы при вводе данных  
2. Средний: обработка спецсимволов, сохранение регистра  
3. Низкий: очень длинный текст

## 6. Баг-репорты
**Оформление багов:**  
- Уникальный ID (BUG-001, BUG-002…)  
- Название  
- Окружение
- Связанные артефакты 
- Шаги воспроизведения  
- Ожидаемый результат  
- Фактический результат  
- Severity
- Priority
- Обоснование

**Хранение:**  
- Markdown-файл `bug_reports.md` в репозитории  

**Приоритизация:**  
- Высокий: падение программы, критические ошибки функционала  
- Средний: визуальные или менее критичные функциональные баги  
- Низкий: редкие или малозначимые баги  

---

## 7. Риски и ограничения
**Риски:**  
- Падение программы из-за некорректного пользовательского ввода
- Ошибки в обработке диапазонов сдвига  
- Потеря регистра или спецсимволов  

**Ограничения:**  
- Нет UI/UX тестирования, т.к. проект консольный
- Нет API или базы данных
