# Требования: "Шифр Цезаря"

---

## 1. Введение

### 1.1 Описание
Приложение реализует алгоритм шифра Цезаря для шифрования и дешифрования текстовых данных с поддержкой русского и английского алфавитов.

---

## 2. Функциональные требования 

### FR-001: шифрование текста
**Описание:**  
Система должна шифровать введённый текст с использованием заданного шага сдвига.

**Входные данные:**
- Текст (строка)
- Шаг сдвига (целое число)

**Выходные данные:**
- Зашифрованный текст

---

### FR-002: дешифрование текста
**Описание:**  
Система должна выполнять обратное преобразование текста.

**Входные данные:**
- Зашифрованный текст
- Шаг сдвига

**Выходные данные:**
- Исходный текст

---

### FR-003: поддержка алфавитов
**Описание:**  
Система должна поддерживать:
- Английский алфавит (A-Z, a-z)
- Русский алфавит (А-Я, а-я)

**Вход:** текст + выбранный алфавит  
**Выход:** корректно обработанный текст  

---

### FR-004: диапазон сдвига
**Описание:**  
Система должна позволять пользователю задавать диапазон значений сдвига.

**Вход:**
- Левая граница (int)
- Правая граница (int)

**Выход:**
- Список результатов для каждого значения диапазона

---

### FR-005: циклический сдвиг
**Описание:**  
Система должна реализовывать циклический сдвиг символов.

**Пример:**
- `z` в `a`
- `я` в `а`

---

### FR-006: сохранение регистра
**Описание:**  
Регистр символов должен сохраняться.

**Пример:**
- `AbC` в `BcD`

---

### FR-007: обработка спецсимволов
**Описание:**  
Цифры, пробелы и спецсимволы не изменяются.

**Пример:**
- `abc!` в `bcd!`

---

### FR-008: отрицательный сдвиг
**Описание:**  
Система должна поддерживать отрицательные значения сдвига.

---

### FR-009: большие значения сдвига
**Описание:**  
Система должна нормализовать шаг сдвига по длине алфавита.

---

### FR-010: смешанный ввод
**Описание:**  
Система должна обрабатывать строки с несколькими алфавитами.

---

### FR-011: поддержка Unicode
**Описание:**  
Unicode-символы (например, emoji) не должны изменяться.

---

## 3. Варианты использования

### UC-001: шифрование текста
**Актор:** пользователь  

**Сценарий:**
1. Пользователь запускает программу  
2. Выбирает режим шифрования  
3. Вводит текст  
4. Вводит шаг сдвига  
5. Получает результат  

**Результат:** зашифрованный текст  

---

### UC-002: дешифрование текста
**Актор:** пользователь  

**Сценарий:**
1. Выбор режима дешифрования  
2. Ввод текста  
3. Ввод шага  
4. Получение исходного текста  

---

### UC-003: некорректный ввод
**Актор:** пользователь  

**Сценарий:**
1. Ввод некорректного шага (например, текст)  
2. Система выводит сообщение об ошибке  
3. Запрашивает повторный ввод  

---

## 4. Пользовательские истории

- Как пользователь, я хочу шифровать текст, чтобы скрыть его содержание  
- Как пользователь, я хочу дешифровать текст, чтобы восстановить исходные данные  
- Как пользователь, я хочу вводить разные алфавиты, чтобы работать с разными языками  
- Как пользователь, я хочу получать сообщение об ошибке при неверном вводе, чтобы исправить данные  

---

## 5. Нефункциональные требования (NFR)

### NFR-001: надежность
Система не должна аварийно завершаться при ошибках ввода.

### NFR-002: производительность
Система должна обрабатывать строки длиной >1000 символов без заметных задержек.

### NFR-003: удобство использования
Интерфейс должен быть понятным (консольный ввод с подсказками).

### NFR-004: валидация
Система должна проверять корректность входных данных.

---

## 6. Ограничения

- Программа работает только в консоли  
- Нет графического интерфейса  
- Поддерживаются только русский и английский алфавиты  
- Нет интеграции с внешними системами  

---

## 7. Связь с тестированием

Каждое требование покрывается:
- тест-кейсами  
- чеклистом  
- баг-репортами
