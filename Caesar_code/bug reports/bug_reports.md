#  Баг репорты — Caesar Cipher


## BUG-001  
### Алгоритм: некорректная работа при отрицательном шаге сдвига (CC-010)

### Окружение
- Dev / Staging  
- Python 3.14  
- Консольное приложение  

### Связанные артефакты
- Тест-кейс: CC-010  
- Чек-лист: дополнительные проверки (отрицательный шаг) 
- Тип тестирования: исследовательский 

### Шаги воспроизведения
1. Запустить программу  
2. Ввести текст `abc` 
3. Выбрать режим шифрования 
4. Выбрать язык
5. Ввести шаг сдвига `-1`  
6. Нажать Enter  

### Ожидаемый результат
- Текст корректно сдвигается в обратную сторону: zab

### Фактический результат
Программа не обрабатывает отрицательное значение  

### Severity
Высокий  

### Priority
Высокий  

### Обоснование
- **Severity:** нарушается основной функционал алгоритма шифрования — программа не выполняет корректное преобразование текста  
- **Priority:** баг затрагивает базовую логику и может встречаться в реальном использовании (пользователь может ввести отрицательное значение), требует срочного исправления.
