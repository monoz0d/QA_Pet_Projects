# Требования: "Числовая угадайка"

---

## 1. Введение

### 1.1 Описание
Приложение реализует игру, в которой пользователь должен угадать загаданное компьютером число в заданном диапазоне. Система подсказывает, больше или меньше введённое число, и отслеживает количество попыток.

---

## 2. Функциональные требования 

### FR-001: генерация случайного числа
**Описание:**  
Система должна случайным образом генерировать число в заданном диапазоне.

**Входные данные:**
- Левая граница диапазона (целое число)
- Правая граница диапазона (целое число)

**Выходные данные:**
- Загаданное число (скрыто от пользователя)

---

### FR-002: ввод числа пользователем
**Описание:**  
Система должна принимать пользовательский ввод и проверять корректность данных.

**Входные данные:**
- Число, введённое пользователем

**Выходные данные:**
- Сообщение о корректности ввода
- Запрос повторного ввода при некорректных данных

---

### FR-003: проверка числа
**Описание:**  
Система должна сравнивать введённое число с загаданным и предоставлять подсказки.

**Входные данные:**
- Число пользователя
- Загаданное число

**Выходные данные:**
- Сообщение "Ваше число больше загаданного" или "Ваше число меньше загаданного"
- Сообщение о победе при совпадении

---

### FR-004: подсчёт попыток
**Описание:**  
Система должна отслеживать количество попыток пользователя.

**Выходные данные:**
- Количество попыток
- Вывод пользователю после окончания игры

---

### FR-005: повтор игры
**Описание:**  
Система должна предоставлять возможность начать игру заново после завершения.

**Входные данные:**
- Ответ пользователя (`да` / `нет`)

**Выходные данные:**
- Перезапуск игры или завершение программы

---

### FR-006: обработка ошибок ввода
**Описание:**  
Система должна корректно обрабатывать:
- буквы вместо числа
- числа вне диапазона
- отрицательные числа
- пустой ввод

**Выходные данные:**
- Сообщение об ошибке
- Запрос повторного ввода

---

## 3. Варианты использования

### UC-001: угадывание числа
**Актор:** пользователь  

**Сценарий:**
1. Пользователь запускает программу  
2. Вводит границы диапазона  
3. Вводит числа для угадывания  
4. Получает подсказки "больше"/"меньше"  
5. Угадал число — получает сообщение о победе  
6. Предлагается повтор игры  

**Результат:** пользователь угадывает число и видит количество попыток  

---

### UC-002: некорректный ввод
**Актор:** пользователь  

**Сценарий:**
1. Ввод некорректного значения (буквы, пустой ввод, число вне диапазона)  
2. Система выводит сообщение об ошибке  
3. Запрашивает повторный ввод  

---

### UC-003: завершение игры
**Актор:** пользователь  

**Сценарий:**
1. После окончания игры пользователь вводит `нет`  
2. Система завершает работу  

---

## 4. Пользовательские истории

- Как пользователь, я хочу угадывать число, чтобы проверить свои способности к логике  
- Как пользователь, я хочу получать подсказки "больше" или "меньше", чтобы корректировать догадки  
- Как пользователь, я хочу, чтобы система не падала при некорректном вводе, чтобы безопасно продолжать игру  
- Как пользователь, я хочу видеть количество попыток, чтобы оценивать эффективность угадывания  
- Как пользователь, я хочу иметь возможность играть заново после окончания, чтобы продолжить игру  

---

## 5. Нефункциональные требования (NFR)

### NFR-001: надежность
Программа не должна аварийно завершаться при ошибках ввода.

### NFR-002: производительность
Система должна обрабатывать диапазоны до 10000 без задержек.

### NFR-003: удобство использования
Интерфейс должен быть понятным и дружественным для пользователя (консольный ввод с подсказками).

### NFR-004: валидация
Система должна проверять корректность всех вводимых данных.

---

## 6. Ограничения

- Программа работает только в консоли  
- Поддерживаются только целые числа  
- Нет графического интерфейса  
- Нет интеграции с внешними системами  

---
