# Тест-план проекта "Числовая угадайка"

## 1. Введение
**Описание проекта:**  
Консольная игра, в которой пользователь должен угадать случайное число в заданном диапазоне. 

**Поддерживаемые функции:** 
- Ввод границы диапазона  
- Генерацию случайного числа  
- Подсказки (больше/меньше)  
- Подсчет количества попыток  
- Повтор игры  

**Цель тестирования:**  
Проверить корректность игровой логики, обработку пользовательского ввода и устойчивость программы к ошибкам.

**Особенности тестирования:**  
- Фокус на пользовательском вводе  
- Проверка циклов и условий  
- Негативные сценарии (невалидный ввод)  

---

## 2. Область тестирования
**Покрываемые модули:**  
- Ввод границы диапазона  
- Генерация случайного числа

- Подсчет попыток

**Исключаем:**  
- UI/UX тестирование  
- Сетевые взаимодействия  
- Работа с файлами  

---

## 3. Тестовая стратегия
**Типы тестирования:**  
- **Функциональное:** проверка логики угадывания числа  
- **Негативное тестирование:** некорректный ввод (буквы, пустые значения)  
- **Граничные значения:** граница = 1, большие значения  
- **Исследовательское тестирование**

**Приоритет тестов:**  
1. Критические: корректность угадывания и отсутствие падений  
2. Средний: обработка некорректного ввода  
3. Низкий: повтор игры

## 6. Баг-репорты
**Оформление багов:**  
- Уникальный ID (BUG-001, BUG-002…)  
- Название  
- Окружение
- Связанные артефакты 
- Шаги воспроизведения  
- Ожидаемый результат  
- Фактический результат  
- Severity
- Priority
- Обоснование

**Хранение:**  
- Markdown-файл `bug_reports.md` в репозитории  

**Приоритизация:**  
- Высокий: падение программы, критические ошибки функционала  
- Средний: обработка повторного ввода 
- Низкий: визуальные или редкие баги  

---

## 7. Риски и ограничения
**Риски:**  
- Падение программы из-за ввода нечисловых значений
- Некорректная граница диапазона  
- Бесконечные циклы при ошибках  

**Ограничения:**  
- Только консоль  
- Нет хранения данных
