# Баг репорты - Числовая угадайка

## BUG-001 - Ввод числа: падение программы при вводе букв

## Подробное описание

При вводе нечислового значения (например, "да") программа завершается с ошибкой.

Согласно тест-плану и тест-кейсу **NG-006**, система должна корректно обрабатывать невалидный ввод.

---

### Требование

Программа должна:

* валидировать пользовательский ввод
* принимать только числовые значения
* обрабатывать нечисловой ввод без падения
* выводить сообщение об ошибке
* повторно запрашивать ввод

---

### Ожидаемый результат

* Программа не падает
* Отображается сообщение о некорректном вводе
* Пользователю предлагается повторить ввод

---

### Фактический результат

Программа завершается с ошибкой:

```text
ValueError: invalid literal for int() with base 10: 'да'
```

---

## Шаги воспроизведения

* Запустить программу
* Ввести границу диапазона (например, 10)
* Ввести "да"
* Нажать Enter

---

## Окружение

* Среда: Dev / Staging
* Язык: Python 3.14
* Тип приложения: консольное

---

## Severity

Высокая

### Обоснование

Падение программы приводит к полной остановке выполнения.

---

## Priority

Высокая

### Обоснование

Частый пользовательский сценарий

## BUG-002 - Ввод числа: падение программы при вводе отрицательного числа

## Подробное описание

При вводе отрицательного числа в качестве границы диапазона программа завершается с ошибкой генерации случайного числа.

Согласно тест-плану и тест-кейсу **NG-007**, система должна валидировать диапазон значений.

---

### Требование

Программа должна:

* проверять корректность диапазона
* принимать только положительные числа
* предотвращать создание некорректного диапазона
* выводить сообщение об ошибке
* повторно запрашивать ввод

---

### Ожидаемый результат

* Программа не падает
* Пользователь получает сообщение о некорректном вводе
* Ввод запрашивается повторно

---

### Фактический результат

Программа завершается с ошибкой:

```text
ValueError: empty range in randrange(1, -2)
```

---

## Шаги воспроизведения

* Запустить программу
* Ввести границу диапазона (например, -3)
* Нажать Enter

---
