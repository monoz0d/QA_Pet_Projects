# Тест-план проекта "Калькулятор систем счисления"

## 1. Введение
**Описание проекта:**  
Проект реализует конвертер чисел между системами счисления от 2 до 16.

**Поддерживаемые функции:**
- перевод числа **в десятичную систему** и **из десятичной системы**
- конвертация чисел в рамках 2, 4, 8, 10 и 16-ричных систем
- корректная обработку цифр и букв (A–F для 16-ричной системы).

**Цель тестирования:**  
Обеспечить корректное преобразование чисел между системами счисления, проверить работу программы с граничными значениями оснований и чисел, а также исключить падение при некорректном вводе.

**Особенности тестирования:**  
- Фокус на функциональности числового конвертера  
- Проверка работы с различными основаниями (2–16)  
- Негативные тесты на некорректный ввод  

---

## 2. Область тестирования
**Покрываемые модули:**  
- Модуль ввода числа  
- Модуль выбора системы счисления  
- Алгоритм перевода в десятичную систему (`calc_10`)  
- Алгоритм перевода из десятичной системы (`calc_n`)  
- Проверка корректности ввода (`input_verify`)  

**Исключаем:**  
- Работа с внешними файлами  
- Сетевые операции и API  
- UI-тестирование (консольная программа)  

---

## 3. Тестовая стратегия
**Типы тестирования:**  
- **Функциональное:** проверка корректного перевода чисел между системами  
- **Негативное тестирование:** проверка реакции на некорректный ввод, пустую строку, буквы/символы в числах  
- **Граничные значения:**  
  - Основание = 2 и 16  
  - Число = 0  
  - Наибольшее корректное число для выбранного основания  
- **Производительность:** проверка работы с большими числами  
- **Исследовательское тестирование** для выявления неожиданных ошибок  

**Приоритет тестов:**  
1. Критические: корректное преобразование чисел, падение программы при вводе некорректных данных  
2. Средний: обработка букв и спецсимволов  
3. Низкий: очень большие числа

## 6. Баг-репорты
**Оформление багов:**  
- Уникальный ID (BUG-001, BUG-002…)  
- Название 
- Подробное описание
- Требование 
- Ожидаемый результат  
- Фактический результат  
- Шаги воспроизведения  
- Окружение
- Severity
- Priority
- Вложения
- Статус
- Связанные артефакты 

**Хранение:**  
- Markdown-файл `bug_reports.md` в репозитории  

**Приоритизация:**  
- Высокий: падение программы, критические ошибки функциональности  
- Средний: обработка букв, спецсимволов, некритичные ошибки перевода  
- Низкий: редкие или малозначимые ошибки  

---

## 7. Риски и ограничения
**Риски:**  
- Падение программы из-за некорректного пользовательского ввода
- Ошибки при переводе чисел с буквами (A–F)  
- Потеря старших разрядов при очень больших числах  

**Ограничения:**  
- Нет UI/UX тестирования, т.к. проект консольный
- Нет API или базы данных
