# Требования: "Калькулятор систем счисления"

---

## 1. Введение

### 1.1 Описание
Приложение реализует калькулятор для перевода чисел между различными системами счисления (2–16) с поддержкой проверки корректности ввода.

---

## 2. Функциональные требования

### FR-001: перевод числа между системами счисления
**Описание:** Система должна переводить число из одной системы счисления в другую.  
**Входные данные:**  
- Число  
- Система исходного числа (целое число от 2 до 16)  
- Система целевого числа (целое число от 2 до 16)  
**Выходные данные:**  
- Результат перевода числа  

### FR-002: проверка корректности ввода числа
**Описание:** Система должна проверять, что введённое число корректно для выбранной системы счисления.  
**Входные данные:**  
- Число  
- Система счисления  
**Выходные данные:**  
- Сообщение об ошибке при некорректном вводе  

### FR-003: обработка пустого ввода
**Описание:** Система не должна падать при пустом вводе числа.  
**Входные данные:** пустая строка  
**Выходные данные:** сообщение о некорректном вводе  

### FR-004: обработка недопустимого основания
**Описание:** Система должна проверять, что основание системы счисления находится в диапазоне от 2 до 16.  
**Входные данные:** основание системы счисления  
**Выходные данные:** сообщение об ошибке при выходе за пределы  

### FR-005: поддержка больших чисел
**Описание:** Система должна корректно обрабатывать очень большие числа без потери точности.  

---

## 3. Варианты использования

### UC-001: перевод числа
**Актор:** пользователь  
**Сценарий:**  
1. Запуск программы  
2. Выбор исходной и целевой системы  
3. Ввод числа  
4. Получение результата  
**Результат:** корректно переведённое число  

### UC-002: некорректный ввод числа
**Актор:** пользователь  
**Сценарий:**  
1. Ввод числа с недопустимыми символами  
2. Система выводит сообщение об ошибке  
3. Запрос повторного ввода  

### UC-003: некорректное основание
**Актор:** пользователь  
**Сценарий:**  
1. Ввод основания вне диапазона 2–16  
2. Система выводит сообщение об ошибке  
3. Запрос повторного ввода  

---

## 4. Пользовательские истории
- Как пользователь, я хочу переводить числа между системами счисления, чтобы работать с разными форматами данных  
- Как пользователь, я хочу, чтобы некорректный ввод числа не приводил к сбою программы  
- Как пользователь, я хочу получать сообщение об ошибке при недопустимом основании системы, чтобы исправить ввод  

---

## 5. Нефункциональные требования (NFR)

### NFR-001: надежность
Программа не должна аварийно завершаться при некорректном вводе числа или основания системы.

### NFR-002: производительность
Система должна обрабатывать числа с большим количеством цифр (>15) без задержек.

### NFR-003: удобство использования
Консольный интерфейс должен быть понятным, с подсказками о вводе числа и системы счисления.

### NFR-004: валидация
Программа должна проверять корректность ввода числа и оснований систем счисления.

---

## 6. Ограничения
- Работа только в консоли  
- Основания чисел ограничены от 2 до 16  
- Поддерживаются только целые числа  
- Нет интеграции с внешними системами  

---
