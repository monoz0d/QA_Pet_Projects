# Баг репорты — Калькулятор систем счисления

## BUG-001  
### Ввод числа: падение программы при вводе букв вместо числа (NSC-004)

### Окружение
- Dev / Staging  
- Python 3.14  
- Консольное приложение  

### Связанные артефакты
- Тест-кейс: NSC-004  
- Чек-лист: негативные сценарии (ввод букв вместо числа)  
- Тип тестирования: негативное  

### Шаги воспроизведения
1. Запустить программу  
2. Выбрать перевод из десятичной системы  
3. Ввести число с буквами, например `2G`  
4. Нажать Enter  

### Ожидаемый результат
- Программа не падает  
- Выводится сообщение о некорректном вводе  
- Пользователю предлагается повторить ввод  

### Фактический результат
- Программа завершает работу с ошибкой:  
```ValueError: invalid literal for int() with base 10: '2G'```

### Severity
Высокий  

### Priority
Высокий  

### Обоснование: 
**Severity**: ошибка приводит к падению программы при любом некорректном вводе числа, блокируя дальнейшую работу пользователя. 

**Priority**: баг легко воспроизводится и связан с основным функционалом, требует немедленного исправления.

## BUG-003  
### Алгоритм перевода: некорректный результат при переводе числа из десятичной в десятичную систему (NSC-009)

### Окружение
- Dev / Staging  
- Python 3.14  
- Консольное приложение  

### Связанные артефакты
- Тест-кейс: NSC-009  
- Чек-лист: граничные случаи (большие числа)  
- Тип тестирования: функциональное  

### Шаги воспроизведения
1. Запустить программу  
2. Выбрать перевод из десятичной системы  
3. Ввести большое число, например `123456789012345`  
4. Выбрать в качестве целевой системы — десятичную  
5. Нажать Enter  

### Ожидаемый результат
- Результат совпадает с исходным числом: `123456789012345`  

### Фактический результат
- Результат не совпадает с вводным числом, возникает несоответствие  

### Severity
Средний  
  

### Priority
Средний  

### Обоснование:
**Severity:** программа не падает, пользователь получает некорректный результат, что снижает доверие к конвертеру.

**Priority:** баг влияет на корректность функционала, но не блокирует использование программы полностью.
