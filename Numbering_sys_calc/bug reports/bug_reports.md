# Баг репорты - Калькулятор систем счисления

## BUG-001 - Ввод числа: падение программы при вводе букв вместо числа

## Подробное описание

При вводе пользователем числа с буквами, например `2G`, программа падает.

Согласно требованиям проекта и тест-кейсу **NSC-005**, программа должна корректно обрабатывать некорректный ввод чисел и не завершать выполнение с ошибкой.

---

### Требование

Программа должна:

* проверять корректность ввода числа
* выводить сообщение о некорректном вводе
* предлагать пользователю повторить ввод

---

### Ожидаемый результат

* Программа не падает
* Выводится сообщение о некорректном вводе
* Пользователю предлагается повторить ввод

**Пример:**

```text
Ошибка: введено некорректное число. Попробуйте снова.
```

### Фактический результат

```text
ValueError: invalid literal for int() with base 10: '2G'
```

## Шаги воспроизведения

* Запустить программу
* Выбрать перевод из десятичной системы
* Ввести число с буквами, например `2G`
* Нажать Enter

## Окружение

* Среда: Dev / Staging
* Язык: Python 3.14
* Тип приложения: консольное

## Severity

Высокий

### Обоснование

Программа падает при некорректном вводе числа, блокируя дальнейшую работу пользователя.

## Priority

Высокий

### Обоснование

Баг легко воспроизводится и связан с основным функционалом, требует немедленного исправления.

## BUG-003 - Алгоритм перевода: некорректный результат при переводе числа из десятичной в десятичную систему

## Подробное описание

При переводе большого числа из десятичной в десятичную систему результат не совпадает с исходным числом.

Согласно требованиям проекта и тест-кейсу **NSC-009**, программа должна корректно обрабатывать большие числа без изменения значения.

---

### Требование

Программа должна:

* корректно обрабатывать перевод числа в любую систему
* обеспечивать точное соответствие результата исходному числу

---

### Ожидаемый результат

* Результат совпадает с исходным числом

**Пример:**

```text
Ввод: 123456789012345
Вывод: 123456789012345
```

### Фактический результат

* Результат не совпадает с вводным числом, возникает несоответствие

## Шаги воспроизведения

* Запустить программу
* Выбрать перевод из десятичной системы
* Ввести большое число, например `123456789012345`
* Выбрать в качестве целевой системы — десятичную
* Нажать Enter

## Окружение

* Среда: Dev / Staging
* Язык: Python 3.14
* Тип приложения: консольное

## Severity

Средний

### Обоснование

Программа не падает, но пользователь получает некорректный результат, что снижает доверие к конвертеру.

## Priority

Средний

### Обоснование

Баг влияет на корректность функционала, но не блокирует использование программы полностью.
